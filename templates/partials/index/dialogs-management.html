Scheduler

Token 刷新设置

控制定时刷新任务的启停、执行策略与邮箱间请求节奏。

关闭后将不会自动执行定时刷新任务。
秒
建议设置为 5-10 秒，避免频繁请求触发 API 限流。
天
建议设置为 30 天，防止 Token 因 90 天未使用而过期。
格式：分 时 日 月 星期，例如 0 2 * * * 表示每天凌晨 2:00。
Forwarding

邮件转发设置

这里配置全局转发渠道；实际是否参与转发，仍以每个账号单独开启的状态为准。

分钟
系统会按该频率扫描已开启转发的邮箱。
分钟
`0` 表示不限制最近邮件范围。
开启后，垃圾箱的新邮件也会进入轮询。
支持多选；不选渠道时不会执行自动转发。
当前未选择任何转发渠道。保存后，即使账号已开启转发，也不会向外发送通知。
SMTP 转发

配置接收邮箱与 SMTP 服务器，用于邮件形式的二次转发。

SMTP

可选。留空时默认使用 SMTP 用户名作为发件人邮箱。
Telegram 转发

使用 Bot Token 与 Chat ID，把新邮件摘要推送到指定 TG 会话。

TG
可选。仅用于 Telegram 转发与测试发送，支持 `http://`、`socks5://`、`direct` / `直连`。
企业微信机器人转发

只填写群机器人的 Webhook 地址，系统会以文本消息推送新邮件摘要。

企微
在企业微信群机器人里复制完整 Webhook 地址即可，无需额外配置 Bot Token 或 Chat ID。
